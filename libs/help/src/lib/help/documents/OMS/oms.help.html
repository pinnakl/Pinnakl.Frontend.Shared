OMS

OMS summary The following features are available in this screen -

Trade Entry

Audit Trail

Pinnakl keeps track of all changes made to a trade. The audit trail includes a timestamp and the name of the person who made the changes.

File Attachment

Users can attach files to trades. This includes research for the security, allocation approval, legal docs etc.

Trade Allocation

Users can allocate the trade quantity into various accounts at the custodians. The screen shows the position in each account before and after the trade.
  1. Pinnakl generates the allocates by clicking on the weighing scale icon
  2. Reset button – Just before the weighing scale icon. It’ll delete all the allocations done so far.
  3. Remaining label – Shows the remaining positions that are not allocated yet.

Broker Panel

  1. The Broker panel is used to manage brokers. You can add, delete, update brokers in this screen
  2. Users can access it by clicking Brokers in the oms screen

Add New Broker

  1. When you click on Add new broker in the main broker screen, it’ll display the form for adding broker.
  2. There are 3 sub-sections in this panel.
  3. Upper section -
    • Broker information section
    • By default, self-clearing option is set to True
    • If self-clearing is changed to false, then you have to specify the clearing broker.
  4. Middle section -
    • Broker Identification section
  5. Lower section -
    • Broker Contact section
    • Email needs to be in proper email format.
    • Phone number can contain special characters like “+ - ( )”
  6. Finally click Save button to save all the changes.

Edit Broker

  1. You can also edit/update a broker.

Delete Broker

  1. It works conditionally. Broker gets deleted if there are no trades dependent on the broker. Otherwise it won’t be deleted.

Trade Allocation

  1. The Allocation panel is used to allocate/divide total positions to different accounts and custodians.
  2. Allocation is the 2nd step. First users book a trade then they allocate it.
  3. Users can access it by clicking Next in trade request panel in oms screen.

Side Panel

  1. It is used to show trade information like transaction type, price, description.
  2. Some information like Folder is required to calculate allocations.
